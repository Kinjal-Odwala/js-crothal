Purchase Order Import

The purpose of Purchase Order Import is to update Purchase Orders from JD. Edward. Set Purchase Order's AppTransactionStatusType from 2 to 3.

Class

Class Name
POImporter

UI Elements

UI Element Field Type Mandatory Source Validation
There is no UI element

Configuration Settings

Configuration Key Description
ConnectionString Connection String used to connect to TeamFin database.
ImportFilePath Complete folder path for Purchase Order import file. This folder is shared with other import processes.
ImportBackupFilePath Complete folder path of back ups for Purchase Order Import files. This folder is shared with other import processes.
ImportFileNamePO Purchase Order Import file name.
ImportAt Time when Purchase Order import is executed

Error Messages and Notifications

Error/Event Message
Import file was not found Purchase Order import file was not found in the specified folder.
No Matching Purchase Order was found. Display message "No Matching Purchase Order with right status was found" when matching purchase order was not found in TeamFin based on PurPurchaseOrders.PurPuroOrderNumber and PurPurchaseOrder.AppTransactionStatusType.
Exceptions Check table AppIEExceptions for exception details (AppIECategory = 3)
Imported Number of Records Check table AppIETransactions for total imported records and execution starting/ending time (AppIECategory = 3).

Dependencies

1. Windows Services must be restarted for new configuration settings to take effect.
2. Exception and Event logging depends on LogWriter class.
