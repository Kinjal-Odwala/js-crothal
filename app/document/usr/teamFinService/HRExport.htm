HR Export

The purpose of HR Export to export employee information to JD. Edward.

Class

Class Name
HRExporter

UI Elements

UI Element Field Type Mandatory Source Validation
There is no UI element

Export Process

(HR Export records are retrieved by stored procedure PayPayrollProcessingHRExport and are stored in table EmpEmployeeGeneralsHRExport weekly right after pay period ends)
1. Determine if it is time to export HR records.
2. Call stored procedure EmpEmployeeGeneralsHRExport to retrieve employee records to be exported based on Pay Priod End date
3. Write data to output file

Status Change Codes

New Hire (N)
When adding a new employee to the system, this will automatically TRigger ONE specific set of records. Interfaces E1 through E10, excluding E2, should TRigger. We will need a ‘Change Status’ code of ‘N’ in field 7 and an ‘A’ in field 25. Nothing needs to be passed in field 26.
NOTE: There are very rare occurrences where a new hire may be inactivated in the same pay period in which they are hired. In such instances, a status code of ‘NI’ can be passed in field 7.

Example:
03C2A2,SSN,06/22/2010,CN8,0000,885885,N,"Terreforte2, Ray",SSN,03/03/1969,M,, 06/21/2010,2,1,12.00,4,,,,,B,00000062TESTCODE11, ,A,,#,02/21/2005,AC,

Rehire (R)
Rehires occur when an employee is taken from a terminated status and made active. For this scenario, a status code of ‘R’ is passed in field 7 and codes of ‘A’ and ‘T’ and in fields 25 and 26 respectively.

Example:
03C2A2,SSN,06/22/2010,CN8,0000,885885,R,"Terreforte2, Ray",SSN,03/03/1969,M,, 06/21/2010,2,1,12.00,4,,,,,B,00000062TESTCODE11, ,A,T,#,02/21/2005,AC,

Inactive
This occurs when an employee is taken from an active status and places in either Inactive, FMLA_LOA, or Leave of Absence statuses. A status code or ‘I’ will pass in fields 7 and 25 and a value of ‘A’ on field 26.

Example:
03C2A2,SSN,06/22/2010,CN8,0000,885885,I,"Terreforte2, Ray",SSN,03/03/1969,M,, 06/21/2010,2,1,12.00,4,,,,,B,00000062TESTCODE11, ,I,A,#,02/21/2005,AC,

Reactivation
This will occur when an employee is brought out of an inactive status back to an active status. A status code of ‘A’ will be passed in fields 7 and 25 and a status code of ‘I’ will be in field 26.

Example:
03C2A2,SSN,06/22/2010,CN8,0000,885885,A,"Terreforte2, Ray",SSN,03/03/1969,M,, 06/21/2010,2,1,12.00,4,,,,,B,00000062TESTCODE11, ,A,I,#,02/21/2005,AC,

Termination (T)
This will occur when an employee is taken from an active or inactive status and changed to a terminated status. A status code of ‘T’ will be passed on field 7 and a ‘T’ and an ‘A’ will be in fields 25 and 26 respectively.

Example:
03C2A2,SSN,06/22/2010,CN8,0000,885885,T,"Terreforte2, Ray",SSN,03/03/1969,M, 06/23/2010,06/21/2010,2,1,12.00,4,,,,,B,00000062TESTCODE11,Resignation,T,A,#,02/21/2005,AC,1400

House Code TRansfer
This will happen when an employee is moved from one house code to another. Nothing is passed on field 7 and status codes of ‘A’ are passed on fields 25 and 26. The most important field here is field 23 which houses the house code in the 18-character sTRing.

Example:
03C2A2,SSN,06/22/2010,CN8,0000,885885, ,"Terreforte2, Ray",SSN,03/03/1969,M, 06/23/2010,06/21/2010,2,1,12.00,4,,,,,B,00010101TESTCODE11,,A,A,#,02/21/2005,AC,1400

Ceridian Company TRansfer
This is the most complex of all moves, since it requires TWO TRansactions to be sent over for a single employee. One must be a termination in the old company and the other record must be sent as a rehire in the new company.

Dependencies

1. Windows Services must be restarted for new configuration settings to take effect.
2. Exception and Event logging depends on LogWriter class.
3. Export records are automatically retrieved by scheduled SQL Job which calls procedure PayPayrollProcessingExportSelectHRExport. This job is started weekly right after each pay period ends.
