Work Orders

The purpose of Work Orders is to create and maintain the list of work orders for each House Code. Work orders are billable or non-billable instructions for performing specific tasks as requested by the client. These tasks could be a one time request or they could be recurring.

When a work order has been created it will have a status of Pending Approval. Before a work order can be made available for payroll hours to be entered against it or for invoicing, the work order must be approved by the Area Manager and the Regional Manger. This will be an email notification to both the Area Manager and the Regional Manager. The email will contain a copy of the work order and a link to approve. A Regional Manager approval will override the need for an Area Manager's approval.

Once a work order has been approved by the Area Manager and Regional manager, payroll hours can be entered against the work order and the work order will be available to invoice. When an invoice has been created against the work order, the work order will then have a status of Closed.

Once a work order has been completed you can only view the work order and no edits can be made.

Save Button NA End user input To save the work order details.
  • Once the work order has been saved the following below will happen.
    1. Work Order Number is generated.
    2. Work Order is placed into a "Pending Approval" status.
    3. Approval Emails will be sent to the Area Manager and Regional Manager. The emails will contain a link to allow either Area Manager or Regional Manager to approve the work order request. A Regional Manager's approval will automatically approve the work order and change the work order status to "Approved".
  • Once a work order has been "Approved" the work order will be available in the work order drop list on the payroll screen that will allow payroll hours to be entered against it.
  • When the work is complete and is ready to be invoiced, the work order can be converted to an invoice.
  • The work order's line items will carry over to the invoice line items.
  • Once an invoice has been created against a work order, the work order's status will be changed to "Closed."

Dependencies

1. Transaction Status Types must be defined in a system look up table (AppTransactionStatusTypes)
2. System Variables must be defined in a system look up table (AppSystemVariables)
3. Units must be defined in a system look up table (AppUnits)
4. House Codes must be defined in a system look up table (HcmHouseCodes)
5. Jobs must be defined in a system look up table (HcmJobs)
6. Jobs must be associated to the House Codes (HcmHouseCodeJobs)
7. Chart of accounts must be defined in a system look up table (FscAccounts)
8. Tasks must be defined in a system look up table (WomWorkOrderTasks)
