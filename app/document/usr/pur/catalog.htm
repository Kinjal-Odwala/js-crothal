Catalogs

The purpose of Catalogs is to provide a means to setup and maintain a list of Catalogs available to the Purchasing module. Each Vendor can have multiple Catalogs and each House Code can be associated to multiple Catalogs. Each Catalog can have multiple Items and Items in the Catalog can be active or inactive.

Dependencies

1. Vendors must be defined in a system look up table (PurVendors)
2. Items must be defined in a system look up table (PurItems)
3. House Codes must be defined in a system look up table (HcmHouseCodes)
