Purchase Orders

Purchase Orders is the area in the system where a user can create new purchase orders, view open/placed/posted orders and also allows the user to preview the purchase orders.

Dependencies

1. States must be defined in a system look up table (AppStateTypes)
2. Transaction Status Types must be defined in a system look up table (AppTransactionStatusTypes)
3. Chart of accounts must be defined in a system look up table (FscAccounts)
4. Fiscal Years must be defined in a system look up table (FscYears)
5. Fiscal Periods must be defined in a system look up table (FscPeriods)
6. Send Method Types must be defined in a system look up table (PurPOSendMethodTypes)
7. Vendors must be defined in a system look up table (PurVendors)
8. Items must be defined in a system look up table (PurItems)
9. Catalogs must be defined in a system look up table (PurCatalogs)
10. House Codes must be defined in a system look up table (HcmHouseCodes)
