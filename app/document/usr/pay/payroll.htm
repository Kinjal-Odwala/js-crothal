Salary & Wages

The Salary & Wages module allows the end users to review and modifiy the payroll entries if necessary via the Daily Payroll Approval functionality. End users can also manage the pay codes available for use when recording weekly payroll details per employee.

The main features of the Salary & Wages module are as follows.
  • If the end user selects to review an entry within a closed/future pay period, the records cannot be modified.
  • If the end user selects to review an entry within the current pay period that has already been Approved and Posted, the record cannot be modified unless an end user with appropriate permissions unlocks the record.
  • End user overrides an incorrect time or adds a missed punch and provides the correct date and time. The system recalculates the gross hours based on the new date and time. If the end user does not want the system to automatically subtract an unpaid lunch period from the employee's gross hours, as identified on the House Code setup screen, the employee clears the Lunch option.
  • For House Codes using Time and Attendance, the end user needs the ability to specify which pay code is the default pay code, as Regular does not always apply. If a default pay code is already defined, the current choice would override the previous selection.
  • When a House Code does use Time and Attendance, entries are typically imported from the Kronos application and are then reviewed and modified if necessary via the Daily Payroll Approval functionality.
  • For House Codes using Time and Attendance, punches from the timeclock(s) are transferred to TeamFin based on an established schedule.
  • When a House Code does not use Time and Attendance, entries are made using the Weekly Payroll functionality.
  • Hours cannot be entered if Payroll is being processed, the Salary and Wages module will be appropriately locked down.
  • End users can also manage the pay codes available for use when recording weekly payroll details per employee. Pay codes can be associated with an individual, a specific shift or all employees.
  • The Daily Payroll Approval functionality does allow for the entry of productive pay codes. Once an individual's time for the day has been approved, it is then visible through the Weekly Payroll mechanism. Those who use Time and Attendance can add/modify non-productive hours and dollar differentials via the Weekly Payroll mechanism but they cannot modify productive hours via the Weekly Payroll screen.

Dependencies

1. Nodes must be defined in a system look up table (HirNodes)
2. Groups must be defined in a system look up table (HirGroups)
3. Units must be defined in a system look up table (AppUnits)
4. Users must be defined in a system look up table (AppUsers)
5. Roles must be defined in a system look up table (AppRoles)
6. Nodes must be associated to the Roles (AppRoleNodes)
7. Groups must be associated to the Roles (AppRoleGroups)
8. Nodes must be associated to the Groups (HirGroupNodes)
9. Persons should be available in a look up table (PplPeople)
10. Transaction Status Types must be defined in a system look up table (AppTransactionStatusTypes)
11. System Variables must be defined in a system look up table (AppSystemVariables)
12. Employees should be available in a look up table (EmpEmployeeGenerals)
13. Work Shifts must be defined in a system look up table (EmpWorkShifts)
14. Fiscal Years must be defined in a system look up table (FscYears)
15. Fiscal Periods must be defined in a system look up table (FscPeriods)
16. House Codes must be defined in a system look up table (HcmHouseCodes)
17. Jobs must be defined in a system look up table (HcmJobs)
18. Jobs must be associated to the House Codes (HcmHouseCodeJobs)
19. Payroll Companies must be defined in a system look up table (PayPayrollCompanies)
20. Pay Frequency Types must be defined in a system look up table (PayPayFrequencyTypes)
21. Pay Codes must be defined in a system look up table (PayPayCodes)
22. Employee Punch Types must be defined in a system look up table (PayEmployeePunchTypes)
23. Invoices should be available in a look up table (RevInvoices)
24. Work Orders should be available in a look up table (WomWorkOrders)
